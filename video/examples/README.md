# examples/ — INSPIRAÇÃO, não template

⚠️ **Estes exemplos NÃO são receitas pra copiar.** Cada vídeo é único — você
**constrói cada um sob medida**, analisando o caso específico:

- **Paleta** — qual cor/contraste combina com a marca/tema (não reuse a cor de um exemplo)
- **Comunicação** — quem é o público? qual a mensagem? o tom (técnico, divertido, premium, social)?
- **Conteúdo** — o que cada cena precisa dizer; o ritmo (calmo elegante vs. rápido no beat)
- **Formato** — 16:9 (README/YouTube), 9:16 (Reels/Stories), 1:1; duração

Use os exemplos pra **descobrir o que é POSSÍVEL** e pegar **técnicas concretas** — não
pra clonar. Veja o vocabulário completo em `references/motion-design.md`.

## showcase/ — os 21 prompts oficiais (remotion.dev/prompts)

Catálogo de referência por categoria. Quando um pedido lembra um caso, leia o prompt
mais próximo pra ver COMO a técnica foi descrita — depois adapte ao caso real.

- **motion-graphics-and-3d/** — cinematic-tech-intro, shape-to-words (morph), pixel-font,
  glitch-logo-3D. Técnicas: pop-in, morph com flubber, 3D metálico, pixel-building.
- **product-and-marketing/** — kinetic-marketing, launch-video-on-x, presscut (recriar UI),
  vvterm (Apple-style), cd-store (counter/cards), skills-announcement (typewriter).
- **content-and-social/** — headline-highlight (rough.js), strava (mapa+rota), CTA-overlay
  (lower-third transparente / prores).
- **maps-and-data/** — bar-line-chart (data viz), rocket-timeline, games-3D (three.js tower),
  travel-map (câmera em rota + landmark 3D).
- **audio/** — audio-spectrum (barras reativas ao som).

Cada `.md` tem o prompt original + autor + ferramenta usada. São pontos de partida de
ideias — a execução final é sempre análise + decisão pro caso à frente de você.
